# Настройка домена nanobanana4k.ru в Vercel

## Проблема
404 ошибки при обращении к домену `nanobanana4k.ru` и `www.nanobanana4k.ru` связаны с неправильной настройкой домена в Vercel.

## Решение

### Шаг 1: Добавьте домен в Vercel

1. Откройте проект в Vercel Dashboard: https://vercel.com/dashboard
2. Перейдите в настройки проекта: **Settings** → **Domains**
3. Добавьте домены:
   - `nanobanana4k.ru` (основной домен)
   - `www.nanobanana4k.ru` (с www)

### Шаг 2: Настройте DNS записи

Vercel покажет вам DNS записи, которые нужно добавить у вашего регистратора домена:

#### Для основного домена (nanobanana4k.ru):
- **Тип**: `A` или `CNAME`
- **Имя**: `@` или оставьте пустым
- **Значение**: IP адрес или CNAME, который предоставит Vercel

#### Для поддомена www (www.nanobanana4k.ru):
- **Тип**: `CNAME`
- **Имя**: `www`
- **Значение**: CNAME, который предоставит Vercel (обычно что-то вроде `cname.vercel-dns.com`)

### Шаг 3: Проверьте настройки домена

После добавления DNS записей:
1. Подождите несколько минут для распространения DNS (может занять до 24 часов)
2. В Vercel статус домена должен измениться на "Valid" или "Configured"
3. Проверьте доступность: `https://nanobanana4k.ru`

### Шаг 4: Настройте редирект (опционально)

Если хотите, чтобы `www.nanobanana4k.ru` редиректил на `nanobanana4k.ru` (или наоборот):

1. В настройках домена в Vercel выберите один как основной
2. Настройте редирект в разделе **Domains**

### Шаг 5: Проверьте SSL сертификат

Vercel автоматически выдаст SSL сертификат для вашего домена. Проверьте:
- Статус SSL в настройках домена должен быть "Valid"
- Доступ по HTTPS должен работать: `https://nanobanana4k.ru`

## Важные замечания

⚠️ **DNS распространение**: Изменения DNS могут занять от нескольких минут до 24 часов.

⚠️ **SSL сертификат**: Vercel автоматически выдаст SSL сертификат через Let's Encrypt. Это может занять несколько минут после настройки DNS.

⚠️ **Проверка DNS**: Используйте инструменты для проверки DNS:
- https://dnschecker.org/
- `nslookup nanobanana4k.ru`
- `dig nanobanana4k.ru`

## Если проблема сохраняется

1. **Проверьте DNS записи**: Убедитесь, что DNS записи правильно настроены у регистратора
2. **Проверьте статус в Vercel**: Домен должен быть в статусе "Valid"
3. **Очистите кеш DNS**: `sudo dscacheutil -flushcache` (macOS) или перезапустите браузер
4. **Проверьте логи Vercel**: Убедитесь, что запросы доходят до проекта

## Альтернативное решение через vercel.json

Если нужно настроить редиректы или специальную обработку домена, можно добавить в `vercel.json`:

```json
{
  "redirects": [
    {
      "source": "/",
      "destination": "/",
      "permanent": false
    }
  ]
}
```

Но обычно это не требуется - Vercel автоматически обрабатывает домены после их добавления.
